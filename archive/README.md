## üí° Archive Maintenance

### Adding to Archive

**When a new version supersedes documents:**
1. Move old version to appropriate archive subdirectory
2. Update this README
3. Create subdirectory README if needed
4. Document in CHANGELOG

**Naming convention:**
- Create version-specific folders: `pre-vX.X.X/`
- Or thematic folders: `_process_history/`
- Keep original filenames for traceability

### Archive Guidelines

**DO archive:**
- Superseded comprehensive documents
- Old process documentation
- Historical completion reports
- Previous version artifacts (when v2.0.0 comes)

**DON'T archive:**
- Current version documents
- Active process documentation
- Living templates
- Up-to-date strategic documents
