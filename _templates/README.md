## Overview

This directory contains templates for creating new documents and maintaining consistency across the Eco Balance Documentation vault. Use these templates to ensure new content follows established patterns and standards.

---

## Available Templates

### Strategic Document Templates

**strategic_document_template.md**
- Template for creating new strategic documents (00-70 series)
- Includes standard sections: Overview, Objectives, Approach, Implementation, Metrics
- Provides consistent structure and formatting

### Implementation Task Templates

**implementation_task_template.md**
- Template for creating new implementation task guides
- Includes: Objective, Prerequisites, Steps, Resources, Timeline, Success Criteria
- Ensures consistent task documentation across all categories

### Version Control Templates

**release_notes_template.md**
- Template for creating version release notes
- Includes: Overview, Highlights, What's New, Statistics, Impact, Migration Guide
- Use when creating new version release notes in `_versions/vX.X.X/RELEASE_NOTES.md`

**changelog_entry_template.md**
- Template for creating changelog entries
- Follows Keep a Changelog format
- Use when adding entries to `CHANGELOG.md`

### Analysis & Reporting Templates

**research_synthesis_report.md**
- Template for research synthesis reports
- Includes: Research Question, Methodology, Findings, Conclusions, Recommendations

**security_audit_report.md**
- Template for security audit reports
- Includes: Scope, Methodology, Findings, Risk Assessment, Recommendations

**stakeholder_review_report.md**
- Template for stakeholder review reports
- Includes: Stakeholder Analysis, Feedback Summary, Recommendations

**strategic_challenge_report.md**
- Template for strategic challenge analysis
- Includes: Challenge Description, Root Causes, Impact Analysis, Solutions

### Planning Templates

**execution_playbook.md**
- Template for execution playbooks
- Includes: Objectives, Phases, Tasks, Resources, Timeline, Success Metrics

**weekly_action_plan.md**
- Template for weekly action plans
- Includes: Goals, Tasks, Resources, Timeline, Success Criteria

**scenario_analysis.md**
- Template for scenario analysis
- Includes: Scenario Description, Assumptions, Outcomes, Recommendations

### Mapping Templates

**dependency_map.md**
- Template for dependency mapping
- Includes: Dependencies, Relationships, Critical Path, Risk Assessment

**empathy_map.md**
- Template for empathy mapping
- Includes: Stakeholder Perspective, Needs, Pain Points, Opportunities

## Usage Guidelines

### When to Use Templates

1. **Creating New Strategic Documents**
   - Use `strategic_document_template.md` for any new 00-70 series document
   - Ensures consistency with existing strategic documents

2. **Creating New Implementation Tasks**
   - Use `implementation_task_template.md` for new task files
   - Maintains consistency across all implementation task categories

3. **Version Releases**
   - Use `release_notes_template.md` when creating new version release notes
   - Use `changelog_entry_template.md` when adding changelog entries

4. **Analysis & Reporting**
   - Use appropriate analysis template for new reports
   - Ensures professional presentation and completeness

### How to Use Templates

1. **Copy the Template**
   - Copy the template file to your target location
   - Rename it appropriately

2. **Fill in the Content**
   - Replace placeholder text with actual content
   - Follow the structure provided
   - Maintain formatting and style

3. **Customize as Needed**
   - Add or remove sections as appropriate
   - Adapt to your specific needs
   - Maintain core structure for consistency

4. **Review and Polish**
   - Check formatting consistency
   - Verify all sections are complete
   - Ensure links and references are correct

## Template Maintenance

### Updating Templates

- Templates should be updated when documentation standards change
- Changes to templates should be reflected in all new documents
- Existing documents don't need to be updated unless major changes occur

### Adding New Templates

- New templates should follow existing patterns
- Include clear purpose and usage instructions
- Add to this README when new templates are created
